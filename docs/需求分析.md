# 需求分析文档

**项目名称**：待定
**版本**：V1.0

---

## 1. 项目概述

开发一个轻量级线上二手物品交易/交换平台，实现用户发布物品、浏览商品、发起交易意向、完成交易的核心闭环。目标用户为有闲置物品需要出售或交换的个人，以及平台管理员。业务范围涵盖物品发布与浏览、交易意向发起与确认、简单站内沟通。

---

## 2. 功能性需求

### 2.1 用户模块
用户可通过手机号或邮箱注册账号，密码经过加密存储。登录采用账号密码方式，返回身份认证令牌。用户可以查看个人资料，包括头像、昵称、注册时间和信用分，也可以编辑修改个人资料中的昵称、头像和联系方式。用户可以查看自己发布的所有物品列表。

### 2.2 物品模块
用户可以发布物品，上传最多五张图片，填写标题、描述、类别、新旧程度、交易方式（买卖或交换），以及期望价格或交换目标。系统提供物品列表页面，按发布时间倒序分页展示，显示封面图、标题和价格。用户可进入物品详情页查看完整信息，包括卖家信息、物品图片和详细描述。发布者可以编辑或下架自己发布的物品。

### 2.3 交易模块
买家可在物品详情页发起交易意向，系统生成交易意向记录。卖家可以确认或拒绝买家的意向，确认后状态变更为交易中。双方协商完成后，卖家可标记交易完成。在交易未完成前，任何一方均可取消交易。交易状态包括待确认、交易中、已完成、已拒绝、已取消。

### 2.4 搜索与筛选
用户可通过关键词对物品标题和描述进行模糊搜索。支持按物品类别进行筛选，也支持按交易方式（买卖或交换）进行筛选。

### 2.5 沟通模块
用户可针对单个物品发送留言，留言内容将关联该物品。用户可以查看某个物品的所有留言，并可以删除自己发送的留言。

### 2.6 管理后台
管理员可以查看所有用户列表，并可禁用或启用用户账号。可以查看所有物品列表，对违规物品进行强制下架处理。可以查看平台的所有交易记录。

---

## 3. 业务规则

需要注意的业务逻辑：

+ 不允许自己购买自己发布的物品
+ 发起意向时需校验买家与卖家不是同一用户
+ 物品下架后不可再发起意向
+ 已下架的物品不显示我想要按钮
+ 同一用户对同一物品只能发起一次意向，防止重复提交。
